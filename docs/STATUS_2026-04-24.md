## 2. System-Snapshot — was heute läuft

### bodenbericht.de (Lead-Magnet, **live**)

- Adresse über Formular oder Quiz eingeben → innerhalb 30–90 Sekunden
  3-Seiten-PDF per Mail
- Satellitendaten aus dem Copernicus-Programm (EGMS, 2019–2023),
  Abdeckung DE/NL/AT/CH
- Ampel grün/gelb/rot plus GeoScore 0–100 plus Karte der Adresse
- Teaser-Copy beschreibt, was der Vollbericht zusätzlich liefert
- Admin-Dashboard mit Lead-Liste, CSV-Export, Ampel/Score-Badge und
  PDF-Download je Lead
- DSGVO-Konformität: Klaro-Consent, Brevo SMTP (EU), keine US-Tools
- Legal Pages vollständig (Impressum, Datenschutz, Widerruf, Datenquellen)

### Backend (FastAPI, PostgreSQL+PostGIS)

- 7,9 Mio. EGMS-Messpunkte in der DB
- Geocoding via Nominatim mit Rate-Limiting (1 Request/Sekunde)
- SoilGrids- und LUCAS-Bodenprobe-Queries
- Chrome-Headless-PDF-Rendering
- Mail-Pipeline mit HTML + Plaintext + eingebettetem Logo
- Stripe-Endpunkte sind im Code, nicht scharfgeschaltet
- Auth-System (JWT) im Code, aber keine UI dafür live
- Sentry-SDK integriert mit PII-Scrubber, wartet auf DSN

### Infrastruktur (Contabo VPS 185.218.124.158)

- Docker Compose: PostGIS 16, Backend (FastAPI), Caddy-Reverse-Proxy
- TLS über Let's Encrypt
- Alembic-Migrationen laufen automatisch
- Landing-Ordner bind-mounted → Änderungen live ohne Rebuild

## 5. Wichtige Befehle

### Deploy nach `git pull`

```
cd /opt/bodenbericht && git pull && docker compose build backend && docker compose up -d --force-recreate backend
```

Alembic-Migrationen laufen automatisch beim Container-Start.

### Admin-Token auslesen

```
grep ^ADMIN_TOKEN /opt/bodenbericht/backend/.env
```

### Diagnose-Scripts

```
bash scripts/check-admin-auth.sh
bash scripts/check-static-map.sh
bash scripts/delete-test-leads.sh          # preview
bash scripts/delete-test-leads.sh --confirm
bash scripts/regenerate-report.sh <email>
```

### Sentry testen (sobald DSN eingerichtet)

```
curl -s -X POST -H "X-Admin-Token: $(grep ^ADMIN_TOKEN /opt/bodenbericht/backend/.env | cut -d= -f2)" http://127.0.0.1:8000/api/_admin/sentry-test
```

Anschließend sollte im Sentry-Dashboard ein RuntimeError-Issue erscheinen,
mit der E-Mail als `[email]@example.test` (PII-Scrubber-Bestätigung).
