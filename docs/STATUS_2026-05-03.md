## 2. Konkrete Bug-Fixes aus dem Review

### `mining_rlp._extract_ampel_risk` Worst-of

Vorher: nur `features[0]` ausgewertet → wenn die WMS-Response ein grünes
Polygon vor einem roten listete, wurde die Adresse fälschlich grün
klassifiziert. Jetzt iteriert über alle Features und gibt die schlechteste
Klassifikation zurück (rot > gelb > gruen).

### `mining_de.search_radius_m=130` raus

Behauptung im PDF („Suchradius 130 m") war eine Lüge — in Wahrheit
Punkt-in-Polygon am Adress-Pixel (~1 m Auflösung). Berechtsame und AAK
sind flächige Polygone, „liegt die Adresse drin?" ist die korrekte
Semantik. Schema entrümpelt, Template auf „Punkt-in-Polygon"-KPI
umgestellt, no-hits-Paragraph gegen die Marketing-Lüge gehärtet.

### `soil_directive.py` toter Code

Eine Zeile `settlement_status = "ok" if X else "ok"` (beide Branches
identisch, nirgends gelesen) gelöscht.

### `bonus_microbial_activity` als 8-€-Leerlauf-Modul

Vorher: `status="not_remote"`, `value=None`, ständig. Jetzt:
SOC-Proxy nach Anderson & Domsch 1989 — Cmic ≈ 2 % von SOC,
qCO2 ≈ 1 μg CO2-C / mg Cmic·h, daraus Basale Atmung ≈ 0,2 × SOC%
[μg CO2-C / g·h]. Bewertungs-Schwellen (Anderson 2003):
≥ 0,4 vital · 0,2-0,4 reduziert · < 0,2 gestresst. Fallback auf
not_remote nur wenn SOC selbst fehlt.

ESDAC Soil Microbial Diversity Map (Cmic + Respiration) bleibt als
Roadmap-Upgrade, Hook in `soil_directive.py` dokumentiert.

## 3. EGMS-Methodik

Distanz-gewichteter Mittelwert für die Ampel-Berechnung. Schema:
`w_i = 1 / max(d_i, 50 m)` über alle PSI im 500-m-Radius. In beiden
Pipelines konsistent (`leads.py` neu eingeführt, `reports.py` von
linear-taper-mit-0.1-Floor auf identisches Schema migriert).

Verhindert False-Positives (Nachbarquartier 400 m weg hat Senkung →
falsch-gelbes eigenes Haus) und False-Negatives (eigener Block senkt
sich, Umgebung stabil → falsch-grünes Haus).

Sanity-Check (5 nahe Punkte 4 mm/yr, 65 entfernte ~0,5 mm/yr):
ungewichtet 0,76 mm/yr → grün, gewichtet 1,28 mm/yr → näher an gelb.
